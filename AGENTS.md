# Концепция работы команды агентов

## Обзор

Система автоматизированной разработки с использованием команды агентов для выполнения задач по плану (PLAN.md).

## Архитектура

```
notes-flow/
├── PLAN.md                 # Основной план проекта со статусами тикетов
├── AGENTS.md              # Концепция работы агентов (этот файл)
├── TICKET-001.md          # Тикеты для агентов
├── TICKET-002.md
├── ...
├── TICKET-009.md
└── (реализованный код)
```

## Процесс работы

### 1. Планирование

- Все задачи определяются в PLAN.md
- Каждый этап реализации имеет свой тикет (TICKET-NNN.md)
- Тикеты имеют приоритет (🔴 высокий, 🟡 средний)
- Статусы: ⏳ pending, 🚧 in_progress, ✅ completed

### 2. Выполнение тикетов

Агент выполняет тикет по следующему алгоритму:

1. **Чтение PLAN.md** — находит первый невыполненный тикет
2. **Чтение тикета** — изучает требования из соответствующего TICKET-NNN.md
3. **Реализация** — создает/модифицирует файлы согласно требованиям
4. **Тестирование** — пишет тесты согласно требованиям в тикете
5. **Запуск тестов** — запускает pytest, убеждается что все проходят
6. **Обновление документации** — обновляет документацию проекта (README.md, AGENTS.md, документацию модулей)
7. **Обновление PLAN.md** — меняет статус тикета на ✅
8. **Git commit** — коммитит изменения
9. **Сообщение** — сообщает о завершении

### 3. Последовательное выполнение

- Агенты работают последовательно
- **После завершения каждого тикета запускается отдельный новый агент для следующего тикета**
- Очередность определяется номером тикета в PLAN.md

### 4. Требования к тикетам

Каждый тикет должен содержать:
- Название этапа
- Описание задачи
- Список компонентов для реализации
- Приоритет
- Требования к тестированию
- Технические детали с примерами кода
- Требования к обновлению документации (если изменения затрагивают публичный API или архитектуру)

### 5. Создание нового тикета (из запроса пользователя)

Когда пользователь просит: **"создай тикет для [описание]..."**

Процесс из двух этапов:

#### Этап 1: Создание сырого тикета (тот же агент)

1. **Создаю файл** `TICKET-NNN.md` где NNN = следующий свободный номер
2. **Вставляю описание пользователя** как есть (без изменений)
3. **Добавляю заголовок и базовую структуру**:
   ```markdown
   # TICKET-NNN: [Название из описания]
   
   ## Описание
   [Описание от пользователя]
   
   ## Компоненты
   - [ ] Список компонентов (из описания)
   
   ## Приоритет
   🔴 высокий / 🟡 средний
   
   ## Статус
   ⏳ pending - требует проработки
   ```
4. **Обновляю PLAN.md**:
   - Добавляю строку в таблицу тикетов со статусом `⏳ pending - требует проработки`
   - Добавляю строку в таблицу этапов со статусом `⏳ pending - требует проработки`
5. **Сообщаю пользователю**:
   ```
   ✅ Тикет TICKET-NNN создан: [Название]
   📝 Тикет требует проработки перед реализацией
   
   🚀 Запусти отдельного агента с командой:
      "проработай тикет TICKET-NNN"
   
   Аген проработает тикет и обновит PLAN.md на статус: 🚧 in_progress
   ```

#### Этап 2: Проработка тикета (отдельный агент)

**Запускается отдельный агент** с командой: **"проработай тикет TICKET-NNN"**

1. **Читает сырой тикет** `TICKET-NNN.md`
2. **Анализирует требования** и определяет:
   - Какие файлы нужно создать/модифицировать
   - Какие компоненты реализовать
   - Какие тесты написать
   - Технические детали с примерами кода
3. **Дополняет тикет**:
   - Добавляет раздел **"Реализация"** с конкретными шагами
   - Добавляет раздел **"Тестирование"** с требованиями к тестам
   - Добавляет раздел **"Технические детали"** с примерами кода
   - Указывает требования к документации
4. **Обновляет PLAN.md**:
   - Меняет статус тикета на `🚧 in_progress`
   - Обновляет таблицу этапов
5. **Сообщает пользователю**:
   ```
   ✅ Тикет TICKET-NNN проработан
   📋 Готов к реализации
   
   🚀 Запусти отдельного агента с командой:
      "реализуй тикет TICKET-NNN"
   ```

### 6. Взаимодействие с пользователем

- После завершения каждого тикета агент сообщает пользователю
- Пользователь может проверить статус через `git log`
- Пользователь может запустить тесты вручную через `pytest`

## Команды для проверки

```bash
# Проверка статуса тикетов
grep "TICKET-" PLAN.md

# Проверка выполненных коммитов
git log --oneline

# Запуск всех тестов
pytest

# Запуск тестов для конкретного тикета
pytest tests/unit/bot/timers/
```
