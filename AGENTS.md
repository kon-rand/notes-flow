# Концепция работы команды агентов

## Обзор

Система автоматизированной разработки с использованием команды агентов для выполнения задач по плану (PLAN.md).

## Архитектура

```
notes-flow/
├── PLAN.md                 # Основной план проекта со статусами тикетов
├── AGENTS.md              # Концепция работы агентов (этот файл)
├── TICKET-001.md          # Тикеты для агентов
├── TICKET-002.md
├── ...
├── TICKET-009.md
└── (реализованный код)
```

## Процесс работы

### 1. Планирование

- Все задачи определяются в PLAN.md
- Каждый этап реализации имеет свой тикет (TICKET-NNN.md)
- Тикеты имеют приоритет (🔴 высокий, 🟡 средний)
- Статусы: ⏳ pending, 🚧 in_progress, ✅ completed

### 2. Выполнение тикетов

Агент выполняет тикет по следующему алгоритму:

1. **Чтение PLAN.md** — находит первый невыполненный тикет
2. **Чтение тикета** — изучает требования из соответствующего TICKET-NNN.md
3. **Реализация** — создает/модифицирует файлы согласно требованиям
4. **Тестирование** — пишет тесты согласно требованиям в тикете
5. **Запуск тестов** — запускает pytest, убеждается что все проходят
6. **Обновление PLAN.md** — меняет статус тикета на ✅
7. **Обновление документации** — обновляет документацию проекта (README.md, AGENTS.md, документацию модулей)
8. **Git commit** — коммитит изменения
9. **Сообщение** — сообщает о завершении

### 3. Последовательное выполнение

- Агенты работают последовательно
- **После завершения каждого тикета запускается отдельный новый агент для следующего тикета**
- Очередность определяется номером тикета в PLAN.md

### 4. Требования к тикетам

Каждый тикет должен содержать:
- Название этапа
- Описание задачи
- Список компонентов для реализации
- Приоритет
- Требования к тестированию
- Технические детали с примерами кода
- Требования к обновлению документации (если изменения затрагивают публичный API или архитектуру)

### 5. Взаимодействие с пользователем

- После завершения каждого тикета агент сообщает пользователю
- Пользователь может проверить статус через `git log`
- Пользователь может запустить тесты вручную через `pytest`

## Команды для проверки

```bash
# Проверка статуса тикетов
grep "TICKET-" PLAN.md

# Проверка выполненных коммитов
git log --oneline

# Запуск всех тестов
pytest

# Запуск тестов для конкретного тикета
pytest tests/unit/bot/timers/
```
