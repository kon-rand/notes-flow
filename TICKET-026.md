## Реализация

### 1. Модуль хранения состояния

**Файл**: `bot/db/backup_state.py`

**Назначение**: Хранение и загрузка состояния последнего бэкапа для каждого пользователя.

**Требования**:
- Использовать JSON файл для хранения состояния
- Не использовать БД
- Хранить только метаданные (timestamp + hash), не полные файлы

**Структура данных**:
```python
from pydantic import BaseModel
from datetime import datetime
from typing import Optional

class BackupState(BaseModel):
    user_id: int
    last_backup_timestamp: Optional[datetime] = None
    last_backup_hash: Optional[str] = None
```

**API**:
```python
class BackupStateManager:
    def save_state(self, user_id: int, timestamp: datetime, hash: str) -> None
    def get_last_state(self, user_id: int) -> Optional[BackupState]
```

### 2. Валидация изменений файлов

**Файл**: `utils/backup_validator.py`

**Назначение**: Проверка, изменились ли файлы пользователя с момента последнего бэкапа.

**Требования**:
- Использовать только временные метки файлов (не хэширование)
- Проверять все файлы: inbox.md, tasks.md, notes.md, archive/*, inbox_backup/*
- Возвращать `True` если есть изменения, `False` иначе

**API**:
```python
def has_changes(user_id: int, last_backup_timestamp: datetime) -> bool:
    """
    Проверяет, изменились ли файлы пользователя с момента последнего бэкапа.
    
    Args:
        user_id: ID пользователя
        last_backup_timestamp: Время последнего бэкапа
        
    Returns:
        True если есть изменения, False иначе
    """
```

### 3. Планировщик бэкапов

**Файл**: `bot/scheduler/backup_scheduler.py`

**Назначение**: Автоматическое создание и отправка бэкапов пользователям.

**Требования**:
- Использовать asyncio для планирования задач
- Поддерживать event-based запуск (после архивации)
- Поддерживать fallback на периодическую проверку (если событие не сработало)
- Не использовать сторонние библиотеки планировщиков

**API**:
```python
class BackupScheduler:
    def schedule_daily_backup(self, user_id: int) -> None
    def check_and_create_backup(self, user_id: int) -> None
    def send_backup_to_user(self, user_id: int, backup_file: bytes) -> None
    def send_notification(self, user_id: int, message: str, file_path: Optional[str] = None) -> None
```

**События**:
- `ARCHIVE_COMPLETED` - отправляется после завершения архивации
- `SCHEDULED_CHECK` - периодическая проверка (fallback)

### 4. Интеграция с архиватором

**Файл**: `handlers/commands.py`

**Изменения**:
- Добавить отправку события `ARCHIVE_COMPLETED` после завершения архивации
- Обработать случай падения архивации (всё равно запустить бэкап)

**Пример**:
```python
@router.message(Command("archive"))
async def archive_handler(message: Message):
    if message.from_user is None:
        return
    user_id = message.from_user.id
    
    today = datetime.now()
    file_manager = FileManager()
    archived_tasks = file_manager.archive_completed_tasks(user_id, today)
    
    await message.answer(f"✅ Архивировано задач за {today.strftime('%Y-%m-%d')}: {len(archived_tasks)}")
    
    # Отправить событие для запуска бэкапа
    await backup_scheduler.emit_archive_completed(user_id)
```

## Технические детали

### Хранение состояния

**Файл**: `data/backup_state.json`

**Формат**:
```json
{
  "123456": {
    "user_id": 123456,
    "last_backup_timestamp": "2026-03-28T02:00:00",
    "last_backup_hash": "abc123def456"
  }
}
```

### Конфигурация

**Переменные окружения**:
```env
# Время проверки (в секундах), если событие не сработало
ARCHIVE_CHECK_INTERVAL=3600

# Количество попыток при ошибке
BACKUP_RETRY_COUNT=3

# Задержка между попытками (в секундах)
BACKUP_RETRY_DELAY=5
```

### Поток выполнения

```
1. Архивация завершается (успешно или с ошибкой)
2. Отправляется событие ARCHIVE_COMPLETED
3. Планировщик получает событие и запускает проверку
4. Получает последнее состояние бэкапа из БД
5. Проверяет изменения через backup_validator
6. Если есть изменения:
   - Создает ZIP бэкап через FileManager.create_backup()
   - Отправляет пользователю: "📦 Ежедневный бэкап создан" + файл
   - Обновляет состояние в БД
7. Если изменений нет:
   - Отправляет пользователю: "ℹ️ Данные не изменились за последние 24 часа"
8. Обновляет timestamp последнего бэкапа в БД
```

## Автоматические ежедневные бэкапы

Бот автоматически создает ежедневные бэкапы ваших данных после завершения архивации.

**Как это работает**:
- Бэкап создается после каждой архивации
- Если файлы не изменились, вы получите уведомление без файла
- Если есть изменения, вам придет полный бэкап
- Бэкапы хранятся только последние (история не сохраняется)

**Настройка**:
- Автоматические бэкапы включены для всех пользователей
- Отключить нельзя (это важная функция безопасности)
